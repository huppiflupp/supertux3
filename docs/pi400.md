# SuperTux3 auf dem Raspberry Pi 400

Der Pi 400 (ARM/aarch64, Raspberry Pi OS auf Debian-Basis, schwache GPU) wird
unterstützt. SuperTux3 erkennt ARM automatisch und schaltet den **Schnell-
Grafikmodus** (nearest-Skalierung statt geglätteter Skalierung) ein.

## Installation
Es gibt keine Python-3.14-Wheels für den Pi – der Code läuft aber auf dem
System-Python 3.11. pygame gibt es fertig aus den Paketquellen:

```bash
sudo apt update
sudo apt install -y python3 python3-pygame git
git clone https://github.com/huppiflupp/supertux3
cd supertux3
bash packaging/install-user.sh        # legt venv an; alternativ direkt:
#   PYTHONPATH=src python3 -m supertux3
```
Falls `install-user.sh` beim venv scheitert (kein Netz für pip), einfach das
System-pygame nutzen:
```bash
PYTHONPATH=src python3 -m supertux3
```

## Performance-Tipps
- Automatisch aktiv auf ARM: `--quality fast`. Manuell erzwingen:
  ```bash
  supertux3 --quality fast --fps 30
  ```
  oder im Spiel: **Optionen → Grafik = „Schnell"**, bzw. in der Pause **G**.
- **Vollbild** (F11 oder Optionen) läuft am flüssigsten, weil das Fenster dann
  exakt integer-skaliert wird.
- Bildrate im Optionsmenü auf **30 FPS** senken, wenn 60 nicht gehalten werden.
- Für maximale Leistung ohne Desktop (KMSDRM, Vollbild direkt auf der Konsole):
  ```bash
  SDL_VIDEODRIVER=kmsdrm supertux3 --fullscreen --quality fast
  ```

## Steuerung
Tastatur (Pfeile/WASD, Leertaste) oder ein an den Pi angeschlossenes USB-/
Bluetooth-**Gamepad** (Stick/D-Pad, A = Springen, Start = Pause). Wird beim
Ein-/Ausstecken automatisch erkannt.

## Bekannte Punkte
- Die internen 960×540 werden auf die Bildschirmgröße skaliert; im Schnellmodus
  ist das für den Pi 400 ausreichend leichtgewichtig.
- Sollte der Ton knacken, in den Paketquellen `pulseaudio`/`pipewire` prüfen –
  SuperTux3 läuft notfalls auch stumm weiter.
